Walk-forward refit
  1. Reserve final holdout (never touch during research).
  2. Use expanding walk-forward (growing train window) — good default when history is moderate and regimes drift slowly.
  3. Chronological splits only (walk-forward or purged K-fold) — no random shuffle.
  4. Fixed rules — no nested hyperparameter CV required (low |Θ|).
  5. Still use chronological OOS; never report full-sample IS as final.
  6. Block-bootstrap OOS returns for confidence bands.
  7. Expanding/rolling walk-forward with fixed rules each step (refit = re-evaluate, not re-tune).
  8. Block-bootstrap OOS returns for confidence intervals.
  9. Regime-stratify OOS returns; stress-test costs (critical for intraday ORB).
  10. One evaluation on final holdout; then forward/paper trade when ready.
