## P1 ‚ÄĒ Bugs & UX

1. **State loss on tab switch.** The Statistics assumptions checklist (`checked`) and glossary detail level (`level`) live in component state, and components unmount on tab change ‚ÄĒ checking 8 boxes then peeking at the Glossary wipes progress. Lift to `App` (like protocol settings already are) or persist to memory.
2. **WFA demo chart mislabeled.** Caption says "Cumulative return" but `WFA_DEMO` values are per-block; and OOS blocks 1‚Äď4 use `0` instead of `null`, so a flat OOS line renders despite `connectNulls={false}`. Use `null` and relabel "per-block return".
3. **Bias chart caption vs scale.** "(OOS fidelity index, 0‚Äď1)" but Y domain is `[0,2]` and the IS series peaks at 1.8. Caption, domain, and series semantics disagree.
4. **No deep linking.** Tab, term, and method state aren't in the URL. Back button does nothing; you can't share a link to a glossary term ‚ÄĒ a real limitation for a reference site. Hash routing (`#glossary/dsr`) is cheap with the existing state lifted in `App`.
5. **FDR definition (`terms.ts` ‚Üí `fwer-fdr`).** `FDR = E[V/R | R>0]` is Storey's **pFDR**. BH controls `FDR = E[V/R ¬∑ 1{R>0}]`. Pedantic, but precision is this site's brand.
6. **Convention violations in own content.** `CV_BIAS.note` wraps the acronym in math (`$CPCV$`) ‚ÄĒ CONTENT.md forbids exactly this; `pbo` glossary math uses `\text{OOS}` subscript where CONTENT.md prescribes `\mathrm{OOS}`.
7. **Copy nit.** In-Sample Backtest "Assumptions: None for validity" reads as "no assumptions needed". Suggest: "No assumptions under which it is valid as OOS evidence."

## P2 ‚ÄĒ Quality & infra

1. **No CI.** Add a GitHub Action: `tsc` + `vite build` + a data-integrity script (term count, duplicate ids, broken cross-references, category membership ‚ÄĒ the exact checks run for this audit). ~30 lines as `scripts/check-data.mjs` + npm script; turns every content PR into a checked PR.
2. **Bundle size.** Single 937 KB minified JS chunk (KaTeX + Recharts + React); Vite warns. Recharts is used on only 2 tabs ‚ÄĒ `React.lazy` the charts; consider `manualChunks` for KaTeX.
3. **No meta description / Open Graph tags** in `index.html`. Sharing the site anywhere produces a bare card. Add description, `og:title/description/image`, `theme-color`.
4. **Unused assets.** `src/assets/hero.png`, `typescript.svg`, `vite.svg` are referenced nowhere ‚ÄĒ delete.
5. **No linter.** Add ESLint (typescript-eslint + react-hooks); CONTENT.md's pre-merge checklist currently relies on `tsc` alone.
6. **A11y gaps.** Sub-tabs use `role="tab"`/`aria-selected` without `tablist` keyboard arrow handling or `tabpanel` ‚ÄĒ incomplete ARIA patterns are worse than none; either finish or drop the roles. Top nav could use `aria-current`. `GlossaryPanel` queries `document.querySelector` instead of a ref.
